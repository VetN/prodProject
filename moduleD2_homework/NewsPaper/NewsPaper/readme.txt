Модель Author:
    - связь "один к одному" со встроенной моделью пользователей User
    - рейтинг пользователя

Модель Category:
    -название категории(уникальность поля unique=True)

Модель Post:
    - связь "один ко многим" с моделью Author
    - поле с выбором: статья или новость
    - автоматическое добавление дата и время создания
    - связь "многие ко многим" с моделью Category (с дополн моделью PostCategory)
    - заголовок статьи/новости
    - текст статьи/новости
    - рейтинг статьи/новости

Модель PostCategory:
    промежуточная модель для связи "многие ко многим"
    - связь "один к многим" с моделью Post
    - связь "один ко многим" с моделью Category

Модель Comment:
    Под каждой новостью/стаьей можно оставлять комментарии, 
    поэтому необходимо организвовать их способ хранения
    - связь "один ко многим" с моделью Post
    - связь "один ко многим" со встроенной моделью User
      (комментарии может оставлять любой пользователь)
    - текст комментария
    - дата и время создания комментария
    - рейтинг комментария

Модели должны реализовывать методы:
    - like(), dislike() в Comment и Post
    - preview() in Post, который возвращает начало статьи(предвар просмотр)
      длиной 124 символа и многоточие в конце
    - update_rating() модели Author, который обновляет рейтинг пользователя,
      переданный в аргумент этого метода.
      Он состоит из:
      - суммарный рейтинг каждой статьи автора умножается на три
      - суммарный рейтинг всех комминтариев автора
      - суммарный рейтинг всех комминтариев к статьям автора

В консоли:
    - создать пользователей
    - объекты модели Author, связанные с пользователями
    - добавить 4 категории в модель  Category
    - добавить статьи и новости
    - присвоить категории ( по несколько категорий в одной стаье/новости)
    - добавить комментарии минимум 4
    - скорректировать рейтинги, используя функции like(), dislike()
    - обновить рейтинги пользователей
    - вывести username и рейтинг лучшего пользователя
      (применяя сортировку и возвращая поля первого объекта) 
    - вывести дату добавления, username автора, рейтинг, заголовок
      и превью лучщей статьи, основываясь на like/dislike к этой статье
    - вывести все комментарии(дата, пользователь, рейтинг, текст) к этой статье
